# MCP Control Center 模块功能、字段、权限与交互规范

## 1. 权限模型

权限按“模块 × 动作”拆分：`查询`、`编辑`、`处理`。处理包括发布、回滚、错误关闭、凭证变更、删除和权限授权。

| 角色 | 查询 | 编辑 | 处理 |
| --- | --- | --- | --- |
| 平台管理员 | 全部模块、全部 Workspace | 全部资产与权限 | 发布、下线、凭证、权限、删除 |
| 产品负责人 | 总览、Service、MCP、调用、知识库 | 描述、Prompt、业务信息、文档 | 提交上架、维护公告 |
| 开发负责人 | MCP、调用、错误、上架、维护 | Schema、启动配置、版本、测试参数 | 沙盒验证、修复错误、回滚草稿 |
| 运维负责人 | 总览、错误、维护、调用 | 告警、服务器、环境、值班人 | Ping、应急下线、错误关闭 |
| 只读成员 | 已授权 Workspace 的全部列表 | 无 | 无 |

页面必须在操作按钮旁显示“可编辑 / 只读”状态；演示账号默认是平台管理员，权限矩阵可切换到其他角色观察界面变化。

## 2. 总览

### 数据维度

时间范围支持近 7 天、近 15 天、近 30 天、近 90 天、近 180 天；切换后指标、趋势、排行和提醒统一变化。Workspace 作为第二维度。

### 指标跳转

- MCP 总数 → MCP 列表
- 已上架服务 → MCP 列表，状态=已上架
- 今日调用次数 → 调用分析，日期=今天
- 平均成功率 → MCP 列表，按成功率倒序
- 平均响应时长 → MCP 列表，按响应时长倒序
- 待处理错误 → 错误中心
- 趋势点 → 调用分析，日期=当天；悬浮显示日期、调用量、成功率、Token
- 调用量排行 → 对应 MCP 详情的“调用分析”Tab
- 描述质量 / AI 评分排行 → 对应 MCP 详情的“概览”Tab

跳转实现要求：日期、排序、状态和 Workspace 不能只改变 URL 文案；必须作为目标页面的初始筛选状态传入，目标页可以继续修改并提供清除筛选操作。

## 3. Service 管理与详情

Service 列表默认是可展开抽屉结构，字段为：名称、状态、调用次数、平均成功率、平均 Token、平均响应时长、应关注 MCP 数量。

展开后的 MCP 行字段为：名称、状态、调用次数、成功率、平均 Token、响应时长、关注原因。点击 MCP 进入详情。

Service 详情页维度：整体描述、业务部门、负责人、产品负责人、技术负责人、包含工具、上下架 MCP、调用总览、错误趋势、版本分布、Workspace 接入情况。调用总览表格的 Service、MCP、日期和错误行均可跳转。

## 4. MCP 列表与详情

列表支持 Workspace、Service、状态、负责人、AI 客户端、调用量、错误率和第三方市场筛选；支持按调用次数、成功率、Token、响应时长排序。

详情支持编辑：产品/开发/运维负责人、工具描述、Prompt、Schema、工作流调用名称、Infra 对用户展示名称和描述、鉴权方式（OpenAPI、用户调用时输入密码、平台托管凭证）。

详情新增“手动测试”Tab：输入参数编辑、执行调用、输出结果、错误模拟、耗时和 Trace；成功和失败都能打开完整上下文。

## 5. 调用分析

以 MCP 调用列表为主，不只展示图表。字段包括日期、MCP、Service、调用次数、成功率、平均响应时长、P95、Token、返回字数、重试次数、AI 客户端。支持日期范围、Workspace、MCP、Service 筛选和多列排序；点击行进入 MCP 调用分析。

## 6. 错误中心

错误中心由总览和错误列表组成，不再放“最近错误日志”和旧的“报错自查”卡片。总览展示错误总数、受影响 MCP、平均解决时长、错误趋势和原因分布。

列表支持日期范围、Workspace、原因分类、错误码、解决状态和解决时长排序。每行展示 MCP、Service、错误数、首次/最后发生时间、解决时长、运维/开发/产品负责人、告警级别。点击告警进入 MCP 详情报错分析 Tab。

错误日志弹窗展示：具体时间、用户与 LLM 对话、调用工具数量、实际参数、返回结果、AI 回复受到的影响、错误上下文版本和一句话影响总结。

## 7. 上架管理

采用可回退的五步向导：

1. 基础配置：名称、Service、Workspace、负责人、标签、风险等级。
2. 技术配置：Transport、启动命令、Schema、超时、鉴权、工作流名称。
3. 沙盒验证：安装地址、健康检查、手动调用、输入输出、错误模拟、验证清单。
4. 审批上架：产品、安全、开发、运维审批；展示意见和待办人。
5. 发布结果：灰度比例、目标 Workspace、发布日志、回滚和维护公告。

每一步都有“上一步 / 保存草稿 / 下一步”，校验不通过时显示字段级原因。

## 8. 服务维护

维护页包含服务器配置、维护人员、告警通知、上线风险日志、服务异常日志、Workspace 环境和鉴权凭证。

- 服务器配置：管理地址、托管地址、代码库、日志、监控、生产/预发地址。
- Ping 测试：选择地址、超时、鉴权模式，显示 DNS、TLS、HTTP、延迟和响应体摘要。
- 异常日志：点击打开详情弹窗，展示影响的 Workspace、Service、MCP、用户数量和处理动作。
- Workspace 环境：每个 Workspace 可配置可见 MCP/Service、沙盒地址和默认环境。
- 凭证：支持 OpenAPI Key、OAuth 2.0、Bearer、AWS SigV4、平台托管密钥；敏感值只展示掩码。

## 9. MCP 市场

条目字段包括分类、公开标签、提供方、一句话介绍、协议（OpenAPI / Anthropic / MCP）、LLM 支持范围、用户 OAuth 需求、平台集中采购、对用户隐藏提供方名称、审核状态和安全评级。

点击条目弹出接入详情：工具清单、权限、数据流、计费方式、授权模式、安装到哪个 Workspace、沙盒安装、健康检查、提交审核。

## 10. 知识库

文档列表字段为标题、分类、版本、负责人、状态、解析进度、关联 MCP、更新时间。支持上传、解析、预览、编辑、删除、搜索、标签、收藏和关联对象。

首版内置文档：MCP 产品设计规范、工具描述与 Prompt 规范、MCP Server 技术接入、鉴权与凭证规范、沙盒验证清单、错误码与排障手册、上架审批流程、运维应急手册。

上传流程为：选择文件 → 上传进度 → 内容解析 → 提取标题/章节/标签 → 关联 Service/MCP → 保存为草稿或发布。

## 11. 权限管理

包含角色列表、成员列表、Workspace 范围、模块权限矩阵和审计日志。每个模块至少配置查询、编辑、处理三列；异常处理需要开发/运维角色，服务器配置需要服务器管理员，凭证需要平台管理员。所有授权变更写入审计日志。
